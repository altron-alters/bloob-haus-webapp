# Citations

Renders Pandoc-style footnotes — the same syntax Obsidian uses natively. Styles the HTML generated by `markdown-it-footnote` using theme CSS tokens.

## Activation

Automatic. Any page containing `[^label]` inline markers activates this visualizer (detected via `.footnote-ref` elements in the rendered HTML). No code fence or frontmatter needed.

## How to write content

Use standard Obsidian/Pandoc footnote syntax anywhere in a markdown file:

```markdown
Some claim worth citing.[^smith2020] Another point.[^jones2019]

[^smith2020]: Smith, J. (2020). *Title of Work*. Publisher. https://doi.org/...
[^jones2019]: Jones, A. (2019). "Article title". *Journal Name*, 12(3), 45–67.
```

- Inline marker `[^label]` — becomes a superscript link in the rendered text
- Definition `[^label]: text` — collected at the bottom of the page in a reference list
- Labels can be any string (no spaces). Numbers, author-year keys, or descriptive slugs all work.
- Definitions can appear anywhere in the file — they are always rendered at the bottom regardless of placement.

## Output HTML

```html
<!-- inline -->
<sup class="footnote-ref"><a href="#fn1" id="fnref1">[1]</a></sup>

<!-- reference section (auto-appended at bottom of page) -->
<hr class="footnotes-sep">
<section class="footnotes">
  <ol class="footnotes-list">
    <li id="fn1" class="footnote-item">
      <p>Smith, J. (2020)… <a href="#fnref1" class="footnote-backref">↩︎</a></p>
    </li>
  </ol>
</section>
```

## Styling

CSS uses only standard theme tokens — no hardcoded values:

| Element | Tokens used |
|---------|-------------|
| Inline superscript link | `--accent-color` |
| Reference section text | `--text-light`, `--font-body` |
| Separator line | `--border-color` |
| Back-link arrow | `--accent-color` |

No new tokens required beyond the standard theme contract.

## Settings

This visualizer has no configurable settings. It is CSS-only — no `browser.js`, no build-time transform.

## Rendering layer

`markdown-it-footnote` runs as a markdown-it plugin at Eleventy build time. Output is static HTML — no JavaScript at runtime. The same plugin can be used in a browser-side markdown-it instance for the future webapp editor.
